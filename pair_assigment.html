2: What’s DOCTYPE declaration meant for?

Solution:

Document type declaration (DTD) is an instruction related to an SGML (Standard Generalized Markup Language) or an XML document (such as a web page) with a document type definition (for example, a specific version definition of HTML). Following the syntax of the document, it is usually presented as a short bookmark. (In general, <! DOCTYPE> is not an HTML tag, it is a declaration statement, used to declare the type of HTML document.)

Therefore, <! DOCTYPE> is not an HTML tag, it is a declaration statement, used to declare the type of HTML document. In HTML documents, <! DOCTYPE> must be placed before the <HTML> tag. According to the layout of HTML pages that do not follow the current HTML5 standards, DOCTYPE will have the task of "sniffing" or "converting", in particular DOCTYPE will determine the layout of the document pages. text / html in any way, such as "quirks mode" or "standards mode". For HTML5, which is an HTML standard not based on SGML, the use of DOCTYPE is only formal. Because browsers' primary task is to execute web pages based on HTML parsing rather than DTD parsing, browsers do not use DTDs and never access them. Moreover, modern browsers are competing fiercely to best support the HTML5 standard, so the DOCTYPE in these browsers can be considered redundant.

4. How is it possible to define the text visible on the top of a browsers window and/or on a tab?

Solution:

The Document of the top level browsing context can be in one of the following visibility states:

hidden

The Document is not visible at all on any screen.

visible

The Document is at least partially visible on at least one screen. This is the same condition under which the hidden attribute is set to false.

The visibility states are reflected in the API via the VisibilityState enum.

On getting, the visibilityState attribute the user agent must run the steps to determine the visibility state:

  • Let doc be the Document of the top level browsing context.
  • If the defaultView of doc is null, return hidden.
  • Otherwise, return the VisibilityState value that best matches the visibility state of doc:
    • Return "visible" if:
      • The user agent is not minimised and doc is the foreground tab.
      • The user agent is fully obscured by an accessibility tool, like a magnifier, but a view of the doc is shown.
    • Return "hidden" if:
      • The user agent is minimized.
      • The user agent is not minimized, but doc is on a background tab.
      • The user agent is to unload doc.
      • The Operating System lock screen is shown.

To accommodate assistive technologies that are typically full screen but still show a view of the page, when applicable, on getting, the [ visibilityState attribute may return visible, instead of hidden, when the user agent is not minimized but is fully obscured by other applications.

7.There are three different ways of adding CSS styles on an (X)HTML page. How? What are the benefits of each way?

  • In-lined: the css script is placed next to the html tag as the value of the style attribute.

    • How: <p> style=”color: red”>hello world </p>
    • Pros: Highest priority means that it can override all the given styles. Quickly to style and good to test the style attribute (reviewing the style is beautiful before put them to the other css file).
    • Cons: Style individually, repeated code if the style apply for many place in project, messy the code, hard to maintain later.

  • Embedded in the head of HTML file (between <style> tag).

    • How: <head><style type=”text/css”> p { color: red }</style></head>
    • Pros: More structure than in-lined style, can apply class or id selector for styling so that write less and more beautiful code.
    • Cons: Only apply for one HTML that CSS is embedded, if you need the same style in other HTML, you have to copy.

  • External style: Write css by it’s own file with extension .css and then includes it in HTML file.

    • How: create css file and style in that. Then link the css file to html file in head tag like : <link href="path/of/css_file.css" rel="stylesheet" type="text/css" />
    • Pros: Code is more structure, css can apply for many html file, easy to maintain and reusable.
    • Cons: a little bit slower developing time than two solutions above, due to create extra files, can be override by In-lined style or Embedded style (in case the embedded code is placed after the external css file in html file).
